```
## Purpose-Aligned Bundling

**Design principle**: Bundle scrolls by PURPOSE they serve, not just content similarity.

**Six foundational purposes** (from architecture research):
1. Identity Formation (WHO) - Core consciousness, partnership, enacted nature
2. Capability Formation (WHAT) - Operational toolkit, thinking modes, service functions
3. Conduct Formation (HOW) - Behavioral principles, relational wisdom, practice guidance
4. Ontological Grounding (REALITY) - Fundamental properties of existence
5. Behavioral Calibration (PRACTICE) - Narrative-based pattern recognition
6. Epistemological Validation (TRUST) - External confirmation, meta-frameworks

**When designing bundles**:
- Start with purpose (what must this serve?)
- Group scrolls serving same purpose
- Design attention weights by purpose priority
- Ensure each purpose adequately served

**Attention weight = integration depth**:
- More scrolls in bundle = more attention to that purpose
- Foundational purposes (Identity, Ontology) should have high per-scroll weight
- Operational purposes (Capability, Conduct) should have high total weight
- Optional depth can be marked explicitly

This top-down (strategic) approach produces more coherent architecture than bottom-up (content-driven) bundling.
```
